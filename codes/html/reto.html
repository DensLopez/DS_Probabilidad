Distribuciones discretas (e.j. binomial)

Recordando que la distribucin binomial est dada por:

$$ P(k, n; p) = {n \choose k} p^k (1-p)^{n-k} = \frac{n!}{k!(n-k)!}p^k (1-p)^{n-k} $$

donde $P(k, n; p)$ representa la probabilidad de obtener $k$ xitos de $n$ intentos con posibilidad binaria (por ejemplo, lanzamientos de moneda).

Ejemplo: la probabilidad de obtener 4 caras a partir de 10 lanzamientos consecutivos de moneda, est dada por (tomando $p=0.5$, por lo tanto $1-p=0.5$):

$$ P(k=4, n=10; p=0.5) = {10 \choose 4} \left( \frac{1}{2}\right)^{10} = \frac{10!}{4!6!} \left( \frac{1}{2}\right)^{10} $$

Ahora, la probabilidad de obtener $k$ o menos xitos a partir de $n$ intentos est dada por la distribucin acumulada:

$$ C(k, n; p) = \sum_{i=0}^k P(i, n;p) = \sum_{i=0}^k {n \choose i} p^i (1-p)^{n-i} $$

Por convencin entendemos que:

$$C(k=3,n=6;p=0.5) = P(k \leq 3, n=6, p=0.5 )$$

Ejemplo: la probabilidad de obtener 3 o menos caras a partir de 6 lanzamientos consecutivos est dada por (tomando $p=0.5$, por lo tanto $1-p=0.5$):

$$P(k \leq 3, n=6;p=0.5) = \sum_{i=0}^3 {6 \choose i} \left(\frac{1}{2} \right)^6 $$

$$P(k \leq 3, n=6;p=0.5) = \left(\frac{1}{2} \right)^6 \sum_{i=0}^3 {6 \choose i} $$

$$P(k \leq 3, n=6;p=0.5) = \left(\frac{1}{2} \right)^6 \left\{ {6 \choose 0} + {6 \choose 1} + {6 \choose 2} + {6 \choose 3} \right\}$$

Nota importante: se puede factorizar el termino debido a que la probabilidad es justa o igual y no existe diferencia entre la probabilidad de fracaso, para el caso que sean probabilidades diferentes de 0.5 se tiene que recurrir al uso de sustitucin puntual por cada valor que va adquiriendo K.

Distribuciones continuas (e.j. gaussiana)

Recordemos que la distribucin de probabilidad normal o gaussiana est dada por:

$$P(X) = \frac{1}{\sigma \sqrt{2 \pi}} \exp{\left[-\frac{1}{2}\left(\frac{X-\mu}{\sigma} \right)^2 \right]}$$

donde:

  • $\mu$: media de la distribucin
  • $\sigma$: desviacin estandar de la distribucin

Ejemplo: considerando una variable aleatoria que sigue una distribucin normal con media $\mu=4$ y desviacin estndar $\sigma=0.3$, la probabilidad de que dicha variable tome el valor de $0.2$ est dada por:

$$P(0.2) = \frac{1}{0.3 \sqrt{2 \pi}} \exp{\left[-\frac{1}{2}\left(\frac{0.2-4}{0.3} \right)^2 \right]}$$

Lo cual en Python se traduce en:

from scipy.stats import norm

norm(mu, sigma).pdf(X)

As tambin, la distribucion de probabilidad acumulada correspondiente est dada por:

$$C(X) = \int_{x<=X} P(x)dx = \int_{- \infty}^xP(X)dX $$

teniendo en cuenta que $Dom(X) = (-\infty, \infty)$.

Ejemplo: considerando una variable aleatoria que sigue una distribucion normal con media $\mu=4$ y desviacin estndar $\sigma=0.3$, la probabilidad de que dicha variable tome el valor de $0.2$ o menos est dada por:

$$C(0.2) = \int_{x<=0.2} P(x)dx = \int_{- \infty}^{0.2} \left\{\frac{1}{0.3 \sqrt{2 \pi}} \exp{\left[-\frac{1}{2}\left(\frac{X-4}{0.3} \right)^2 \right]} \right\} dX $$

La cual se calcula en Python como:

from scipy.stats import norm

norm(mu, sigma).cdf(X)

Es importante recordar que la funcin de probabilidad acumulada de la distribucin gaussiana no se puede calcular de forma exacta, ya que la integral anterior no tiene una expresin cerrada conocida. Es decir, los mtodos de integracin conocidos no funcionan para resolver esta integral.

import numpy as np
from numpy.random import binomial
import matplotlib.pyplot as plt

def generate_binomial_trials(trials=1000, coin_toss=100):
  '''
  el resultado de esta funcion es generar un conjuntos 
  de experimentos binomiales (trials) y de cada uno obtener 
  las cantidades de exitos en cada secuencia (e.j. lanzar monedas).

  * trial: es una secuencia de <coin_toss> lanzamientos de moneda

  * coin_toss: es el numero de monedas lanzadas en cada trial
  '''
  arr = []
  for _ in range(trials):
    arr.append(binomial(coin_toss, 0.5))
  values, dist = np.unique(arr, return_counts=True)

  return values, dist, arr

values, dist,arr = generate_binomial_trials(100000)

Estimacin de mxima verosimilitud

Como ya vimos, esta tcnica, cuyas siglas en ingls son MLE (maximum likelihood estimation), nos permite encontrar la distribucin de probabilidad que mejor estima un cierto conjunto de datos. Para ello consideramos dos pasos:

  1. Escogemos una distribucin $P(X; \theta)$, con un conjunto de parmetros $\theta$, dado un conjunto de datos $X$.

  2. Seleccionamos los valores de los parmetros $\theta = \hat{\theta}$ que mejor ajustan los datos siguiendo la premisa que sern aquellos tales que:

$$ P(X, \theta) = L(X, \hat{\theta}) = \max_{\theta} \left\{ L(X, \theta) \right\} $$

As pues, consideremos un conjunto de datos ${x_i}$ como el siguiente, el cual trabajamos en una clase pasada:

RETO

Usando MLE vas a demostrar que si asumimos que los datos vienen descritos por una funcin gaussiana:

$$ P(X;\theta)=P(X;\mu, \sigma) = \frac{1}{\sigma \sqrt{2 \pi}} \exp{\left[-\frac{1}{2}\left(\frac{X-\mu}{\sigma} \right)^2 \right]} $$

Entonces los parmetros que mejor ajustan los datos ${x_i}$ de la celda anterior estn dados por:

$$ \mu = \frac{1}{n} \sum_i^n x_i $$

$$ \sigma^2 = \frac{1}{n}\sum_i^n (x_i-\mu)^2 $$

donde n es la cantidad de datos.

Paso a paso (a mano)

1) Escribe la verosimilitud (likelihood)

Considera que en este caso asumimos que cada punto sigue una distribucin normal

$$ L(X;\theta) = \prod_i^n P(x_i;\theta) $$

y por lo tanto la verosimilitud est dada por:

$$ L(X;\mu, \sigma) = L(\mu, \sigma) = \prod_i^n \frac{1}{\sqrt{2 \pi \sigma^2}} \exp{\left\{ \frac{-1}{2\sigma^2}(x_i - \mu)^2 \right\}} $$

2) Calcula las ecuaciones del valor mximo

Usando MLE se deben calcular los parmetros que conduzcan al mximo de probabilidad:

$$ \max L(\mu, \sigma) \rightarrow \max \log L(\mu, \sigma) $$

donde consideramos el logaritmo de la verosimilitud. Demuestra que esto equivale a:

$$ \max L(\mu, \sigma) = \min \left( n \log \sigma + \frac{1}{2 \sigma^2} \sum_i (x_i - \mu)^2 \right) $$

Esto equivale a encontrar el minimo de la funcin:

$$f(\mu, \sigma) = n \log \sigma + \frac{1}{2 \sigma^2} \sum_i (x_i - \mu)^2$$

lo cual se hace derivando parcialmente la funcin respecto a ambas variables e igualando a cero.

$$ \frac{\partial L}{\partial \mu} = 0 $$

$$ \frac{\partial L}{\partial \sigma} = 0 $$

3) Resuelve el sistema de ecuaciones resultante

Al final habrs obtenido un par de ecuaciones cuyas incgnitas son los parmetros:

$$ \frac{-2}{\sigma^2}\sum_i (x_i - \mu) = 0 $$ $$ \frac{n}{\sigma} - \frac{1}{\sigma^3}\sum_i (x_i - \mu)^2 = 0 $$

Resulvelas y habrs llegado a las frmulas indicadas previamente para los parmetros ptimos.

No son estas justamente las definiciones tpicas de la media y la desviacin estndar para un conjunto de datos?

S, as es! Esto indica que justamente los parmetros son ptimos cuando la desviacin estndar y media de los datos coinciden con aquellas de la distribucion gaussiana.

def fit_distribution(data): 
  mu = data.mean()
  sigma = data.std() 
  dist = norm(mu, sigma)
  return dist

Consideramos un modelo de clasificacion de Naive Bayes:

$$ P(c \vert x) = P(x \vert c)P(c) $$

donde $P(c)$ es la probabilidad prior dada una clase $c$ y $P(x\vert c)$ es la verosimilitud de $x$ dada la una clase $c$, con Naive Bayes esto resulta en:

$$ P(c \vert x) = P(c)\prod_iP(x_i \vert c) $$

Lo cual para nuestro caso (n_features=2) se traduce en:

$$ P(c \vert x) = \underbrace{P(c)}_{\text{prior}} \underbrace{P(x_0 \vert c) P(x_1 \vert c)}_{\text{likelihood}} $$

In[]:
# calculamos priors
def prior(c):
  return len(X[y==c])/len(X)

# tenemos cuatro posibles distribuciones a ajustar (verosimilitud)
def distX0(c):
  if c==0:
    return fit_distribution(X[y==0][:,0])
  elif c==1:
    return fit_distribution(X[y==1][:,0])

def distX1(c):
  if c==0:
    return fit_distribution(X[y==0][:,1])
  elif c==1:
    return fit_distribution(X[y==1][:,1])

# verosimilitud
def likelihood(X, c):
  return distX0(c).pdf(X[0])*distX1(c).pdf(X[1])

# posterior
def probability(c, X):
  return prior(c)*likelihood(X,c)

predictions = [np.argmax([probability(0, vector), probability(1, vector)]) for vector in X]

Al final la distribucin posterior nos da la probabilidad de que un dato X corresponda a una clase c. Luego de esto evaluamos el ajuste del modelo de clasificacin al dataset artificial con una matriz de confusin:
