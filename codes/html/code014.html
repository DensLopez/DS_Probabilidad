MLE como base para la regresin logstica

Consideramos el problema de MLE:

$$ \max \sum_i \log P(y_i \vert x_i; h) $$

donde:

$y_i$: clase o categora de cada elemento y $x_i$: son los atributos de cada elemento, donde adems cada elemento del dataset satisface una distribucin de Bernoulli:

\begin{equation*} \small P \; \left \{ \begin{array} {lcl} P & y=1 & \text{cuando el resultado es exitoso.} \\ 1-P & y=0& \text{cuando el resultado no es exitoso.} \end{array} \right. \end{equation*}

En este caso la verosimilitud est dada por:

$$ L = \hat{y}y+(1-\hat{y})(1-y) $$

Esta funcin da como resultado probabilidades altas cuando $\hat{y} \sim y$.

Vamos a construir una grfica tridimensional de 2 variables de verosimilitud. Recordemos que:

  • $y_i:$ Categora o atributo del dataset
  • $x_i:$ Dato de la categora.
  • $h:$ Hiptesis de modelamiento (atributos) - Sigmoide

Vamos a realizar una funcin en Python donde $\hat y = yp$ y es correspondiente a la funcin sigmoidal

In[]:
#Definiendo funcin L
def likelihood(y,yp):
    return yp*y + ((1-yp)*(1-y))

Lo que queremos es visualizar la funcin para tener una claridad de lo que esta funcin representa.

Explicacin

Vamos a realizar una grfica de la funcin likelihood(y,yp), como se puede observar es una funcin de 2 variables y como es usual en el clculo de funciones de 2 variables, tenemos que hacer una grfica de 3 dimensiones:

donde los ejes $(x,y)$ son el plano horizontal y el eje vertical es $z$ as $z=$ valores de la verosimilitud.

Lo que representa la grfica es que nuestra funcin de verosimilitud tiene mximos justo donde los necesitamos. Es decir cuando data set $y$ y data del modelo $\hat y$ (0,0) y (1,1). Dando las probabilidades altas o las mas altas, justo cuando ambas $(y - \hat y)$ coinciden. La visualizacin justifica este anlisis

Considerando $p\rightarrow \log(p)$ y sumando la verosimilitud para todos los puntos del dataset obtenemos:

$$\max L = \min (-L)$$ En otras palabras; maximizar quiere decir minimizar el negativo de Verosimilitud. De manera al proceso de minimizar $-L$ es optimizar esa funcin en un problema de clasificacin y se conoce como Cross Entropy - Entropa Cruzada. Y se define como: $$CE = \underbrace{\quad-\sum_i {\left[\: y_i \log \hat {y_i}+(1-y_i)\log(1-\hat{y_i})\: \right]}\quad}_{\text{Costo }\rightarrow \text{ Clasificacin}} $$

Regresin Logstica con Scikit Learn

Recordemos que:

$$ \hat{y} = \frac{1}{1-\exp{(-\text{log-odds})}} $$

donde:
$\text{log-odds} = \beta_0x_0 + \beta_1 x_1 + \dots \beta_n x_n$ y los betas son los parmetros del modelo.

Comprobado, el siguiente paso es construir el modelo.

La idea es que la funcin que llamamos Logistic Regression la declaremos para hacer un modelo de clasificacin, donde el random_state es un generador aleatorio que inicializa a las variables del modelo y solver=liblinear es el mtodo de optimizacin del modelo.

Todo esto es para encontrar la mejor combinacin de parmetros, y despus le decimos al modelo que se ajuste a los datos. cuales datos? los de las 2 primeras clases x[:100],y[:100]. Con esto el modelo quedara listo, porque fue ajustado con el fit()

In[]:
#Ajustando y creando modelo de clasificacin
clf = LogisticRegression(random_state=10, 
    solver='liblinear').fit(X[:100], y[:100])
